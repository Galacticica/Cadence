Language design · esolang

Cadence.

A programming language whose source code is a drum performance. There is no text format — a program is a .wav file of drums being played. One measure of 4/4 is one instruction. This page decodes how.

Anatomy of a measure

Every code measure is 16 sixteenth-note steps split into three fields. The bar below is a real instruction: LOADI R1, 72 — kick opens the bar (data family, no follow-up hit → load immediate), the high tom selects register 1, and the back half of the bar plays the binary rhythm of 72. Hover any step to read its role, then press play to hear the instruction.

Hover a step…

Decoration is comments. Closed hi-hat on steps 1–8 is ignored by the parser — pure groove. A measure with no opcode instrument on step 1 is an entire bar of free decoration. Programs can breathe.

Numbers are rhythms

The operand field (steps 9–16) spells an 8-bit number, most significant bit first: snare = 1, closed hi-hat = 0 — and a rest also counts as 0; the hat is just the audible zero, easier to perform steadily. Each step carries a place value, so the rhythm literally is the number. Toggle the bits below (it starts on the 72 from the bar above) and hear what your number sounds like.

literals max out at 255 — registers are unbounded; build bigger values with arithmetic (LOADI small, then MUL/ADD)

The instruction set

An opcode is a sequence of drums struck in the opcode field — which drums, in what order. Step 1 picks the family (kick, snare, ride, or crash); the follow-up hits pick the variant, and where they land within steps 2–4 is the performer's choice. Four families, 21 opcodes. The mini-grids show each sequence.

Register addressing — the tom fill

A register reference is one or two tom hits read left to right. Four single hits reach R1–R4; a floor-tom-led pair reaches R5–R8 — the high registers literally take a mini tom fill. Destination lives on steps 5–6, source on steps 7–8.

From audio to execution

The .wav is the canonical source. Every program opens with four stick clicks — a count-in the decoder uses to lock tempo and grid phase, like a modem handshake performed by a drummer.

When IN executes, the machine pauses and listens. Drum your number in binary with the buttons — snare = 1, hat = 0, crash submits — or just type it. Either way, this three-bar program squares whatever you give it.

A complete program: countdown

Seven written measures that print 3 2 1. Loops are repeat signs — REPEAT_WHILE R1 replays its block while R1 ≠ 0, exactly like a drummer reading repeats off a chart. Pressing play performs the execution, not the score: the loop comes back around until R1 hits zero, and the console fills in as it runs.

$ cadence run countdown.wav waiting — press play

Hello, World! is 13 characters × (LOADI + OUTC) + HALT = 27 measures — a 54-second drum solo whose output is a greeting. This is the correct amount of ridiculous.

Skips & repeats, visually

There is no if instruction — there are skips. A skip blinds the machine for exactly one code measure. This program reads a number and prints E or O: SKIPNZ guards the even branch, SKIPZ guards the odd branch, and exactly one of the two loads runs. Flip the input and watch which bars survive.

When a branch needs several bars, REPEAT_START 1 is the curly brace: a block that plays exactly once. Skipping an opener skips its entire block — so one skip guards all four measures below.

And here's how the countdown program above actually executes: REPEAT_END always jumps back to its opener, which re-reads R1 — on the fourth check it sees 0 and jumps past the END to HALT.

Proof of concept — Hi!

The whole idea, end to end: seven measures of drums that print Hi!. Press run and watch the console — each character appears at the exact moment the playhead reaches its OUTC bar, at whatever speed the tempo dial says. The output is performed.
